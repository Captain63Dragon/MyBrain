## Philosophy
MyBrain is a personal information OS on Windows. The filesystem is a dumb block store.
Neo4j is the authoritative layer. Files live on disk. Meaning lives in the graph.
The node owns its truth. The file is the delivery mechanism. Once value is extracted
into the graph, the file has done its job.

### Walled Garden
- **Wild** — unvetted, Dropbox/camera rolls. Prompts ingestion.
- **Garden** — curated, `E:\` subfolders. Ingested, named, backed up.
- **Node** — ground truth. Relationships, annotations. File-independent.

### The Divide
Browsers cannot access local filesystem. Docker has limited, explicitly mounted Windows access.
This is architecture, not a bug.

## MFI Protocol
File-based messaging bridging Docker and Windows. YAML files (.mfi), written atomically (.mft → .mfi).
```
/mfi/pending/     ← UI out-box, Windows in-box
/mfi/processing/  ← atomic lock
/mfi/completed/   ← Windows out-box, UI in-box (mfi_broker sole owner)
```
Intent field is cargo — Windows handlers ignore it. Flask result processor branches on it.
CopyMFI intents: `insitu_copy`, `master_source`, `master_target`
MoveMFI intents: `move`, `archive`

## Graph Schema

### Pipeline nodes
```
(MetaFileNode)-[:ORCHESTRATES]->(Dispatch)
(Dispatch)-[:RESULTED_IN]->(OSResult)
(Dispatch)-[:CREATED]->(FileNode)
(FileNode)-[:INSITU_COPY_OF]->(FileNode)
(FileNode)-[:COPY_OF]->(FileNode)
(MetaFileNode)-[:LOGGED]->(f:FileNode:DocYAML)
(MetaFileNode)-[:HAS_FORMAT]->(UIDefinition)   ← designed, not built
```

### Todo/queue nodes
```
(:Todo)   — Vera's domain
(:Idea)   — Vera's domain
(:User:Person {user-id: 'user-owner', handle: 'owner'})  ← placeholder, biometrics TBD

(:Persona)-[:MADE_TODO]->(:Todo)              ← channel signal to Vera (Rex only)
(:User)-[:ASSIGNED]->(:Todo)                  ← Vera's domain, Rex never touches
(:Todo)-[:FOLLOWS_FROM]->(:Todo)              ← sequencing
(:Todo)-[:BLOCKED_BY]->(:Todo)
(:Idea)-[:EVOLVED_INTO]->(:Todo)
(:User)-[:NUDGED {count, last_asked}]->(:Todo)
```

### Todo properties
`todo-id, description, status, priority, owner, friction, created, due, closed, source_pin`

### Friction values
`none, phone-call, difficult, waiting-on-other, hated-meeting`

### Key node types
- **MetaFileNode** — policy engine, schema, patterns, garden path
- **FileNode** — `FileNode` + MFN type labels (e.g. `BusinessCard`)
- **Dispatch** — `mfi-id` is the key
- **OSResult** — linked via `RESULTED_IN`, has `nodes_created`
- **Persona** — Claude persona definitions
- **UIDefinition** — planned, replaces `summary: true` on MFN fields

### MFN Garden Paths
```
BusinessCard → E:\BusinessCards
R2HOdo       → E:\R2Hope
```
`MFN-id` in graph, `META-FILE-NODE` in YAML — known inconsistency, deferred.

## Bot Library Design (see todo-vera-bot-library-scaffold)
Pattern established 2026-03-18 Meta session. Every persona bot function:
```python
PERSONA_FN = "vera.todos.get_pending"   # dotted namespace library ID

def function_name(session: Session, reason: str = "", log_call: bool = True) -> list[dict]:
    """
    Called by: Vera
    Use case: one-liner
    Library ID: vera.todos.get_pending
    Intent for future Vera: context for next session reading the log
    Logging: pass reason= to log intent. Omit or log_call=False to suppress.
    """
    if log_call and reason:
        log.call(PERSONA_FN, reason=reason)
    ...
```
Higher-order bots log once at their level. Constituents pass `log_call=False`.
Reason strings are ore — post-analysis surfaces call chain patterns → new bots.
See terms file for reason vocabulary conventions.
